# ORB Tactical Alert Dashboard

A real-time Opening Range Breakout (ORB) monitoring and recommendation interface designed for Indian indices and equities. This system automates morning volatility monitoring, generating precise entry, target, and stop-loss levels mathematically derived from the opening range boundaries.

## In a Nutshell

The ORB Tactical Alert Dashboard operates on a pure mathematical breakout engine:

1. **Range Definition**: The system monitors price ticks for watched symbols between 9:15 AM and 9:30 AM IST to establish the absolute opening range high and low.
2. **Locking Phase**: At 9:30 AM IST, the boundaries are locked and the midpoint of the range is calculated to serve as the structural stop-loss.
3. **Breakout Detection**: Any subsequent tick crossing above the range high triggers a Bullish Alert. Any tick crossing below the range low triggers a Bearish Alert.
4. **Risk Management**: For every breakout, the system calculates levels with a strict 1:2 risk-to-reward ratio:
   - **Entry Price**: The exact breakout threshold price.
   - **Stop-Loss (SL)**: The opening range midpoint.
   - **Target**: Entry Price +/- (2 * (Entry Price - Stop-Loss)).
5. **Real-time Streaming**: Alerts are instantly persisted to a SQLite database and broadcast over WebSockets to a responsive frontend client.

## Project Structure

- **backend/**: FastAPI backend service implementing the async ORB engine, SQLite database interface, REST API endpoints, and WebSocket broadcasting.
- **frontend/**: React single-page application dashboard constructed with high-end, responsive dark styling for alert viewing and manual logging.
- **tests/**: Unit tests written in Pytest to validate ORB calculation boundaries, breakout detection correctness, and database interactions.

## Quick Start

### Backend Installation

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Create and activate a Python virtual environment:
   ```bash
   python -m venv venv
   .\venv\Scripts\activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Run validation tests:
   ```bash
   python -m pytest ../tests/test_orb_logic.py -v
   python test_synthetic.py
   ```
5. Start the server:
   ```bash
   python run_backend.py
   ```

The backend API will run on `http://localhost:8000` and the WebSocket endpoint will be available at `ws://localhost:8000/ws/alerts`.

### Frontend Installation

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install Node packages:
   ```bash
   npm install
   ```
3. Start the development server:
   ```bash
   npm start
   ```

The dashboard will open on `http://localhost:3000`.
